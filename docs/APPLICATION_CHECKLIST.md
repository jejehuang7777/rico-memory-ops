## Minimum application requirements

- public GitHub profile
- public repository URL
- maintainer identity:
  - primary maintainer
  - or core maintainer
- short explanation of why the repository matters

## Stronger signals

- a clear README
- obvious target users
- examples that work without private context
- signs of adoption:
  - stars
  - downloads
  - issues
  - forks
- a credible explanation of ecosystem value even if stars are still low

## User-owned setup

- choose the public GitHub account
- create the public repository
- decide which email tied to ChatGPT will be used
- if applying for API credits, provide the OpenAI Organization ID

## Draft positioning

This project is not a private memory dump.

It is a reusable toolkit for:

- memory object design
- recall routing
- continuity handoff
- separating hot working state from durable memory

## Submission reminder

Do not apply with a private-only archive.

Apply with a public repo that already shows:

- what problem it solves
- who it helps
- what artifacts are actually reusable
